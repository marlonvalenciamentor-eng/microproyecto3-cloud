1. Analisis de Requerimientos

1.a. Descripcion de la Empresa y sus Necesidades

RetailCo S.A.S. es una cadena colombiana de tiendas de tecnologia con presencia en Bogota, Medellin y Cali. Cuenta con mas de 50.000 clientes registrados y genera aproximadamente 200 transacciones diarias.

El equipo de marketing identifico que un 23% de clientes activos abandona cada ano sin señal previa, generando perdidas de COP $480 millones anuales.

Necesidad: Predecir con antelacion que clientes abandonaran la plataforma para ejecutar campanas de retencion personalizadas.

Requerimientos Funcionales

2.c. Descripcion de Componentes

Select Columns in Dataset

Excluye "customerID" — identificador unico sin valor predictivo que causaba error de demasiados valores unicos en normalizacion.

Clean Missing Data

Elimina filas con valores nulos en modo "Remove entire row" aplicado a todas las columnas.

Edit Metadata

Marca "Churn" como etiqueta (variable objetivo) del problema de clasificacion binaria.

Normalize Data

Aplica MinMax scaling a columnas numericas, escalando al rango [0,1].

Split Data

Division 80/20 con semilla 42 para reproducibilidad.

Two-Class Boosted Decision Tree

Clasificador binario basado en boosting. Robusto con datos tabulares mixtos y desbalance de clases.

Train Model

Entrena el algoritmo con el 80% de los datos. Produce modelo serializado para predicciones.

Score Model

Aplica el modelo al 20% de prueba. Genera probabilidad de churn (0.0-1.0) por cliente.

Evaluate Model

Calcula AUC, Accuracy, Precision, Recall y F1 Score comparando predicciones con valores reales.

4. Conclusiones

  • Azure Machine Learning Designer permite construir pipelines de ML completos visualmente sin escribir codigo, reduciendo el tiempo de implementacion significativamente.
  • El modelo Two-Class Boosted Decision Tree alcanzo AUC de 0.855, superando el objetivo del 80% definido en los requerimientos.
  • La exclusion de identificadores unicos como "customerID" es critica — su inclusion provoca errores de codificacion en la normalizacion.
  • El costo estimado de USD $75.65/mes esta dentro del presupuesto de USD $200, haciendo la solucion economicamente viable.
  • Azure ML ofrece trazabilidad completa de experimentos, versionado de datasets y modelos, facilitando la gobernanza del proceso de ML.
