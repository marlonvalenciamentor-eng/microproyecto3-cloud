1. Analisis de Requerimientos

1.a. Descripcion de la Empresa y sus Necesidades

Descripcion de la Empresa

RetailCo S.A.S. es una cadena colombiana de tiendas de tecnologia con presencia en Bogota, Medellin y Cali. Cuenta con mas de 50.000 clientes registrados en su plataforma de fidelizacion y genera aproximadamente 200 transacciones diarias a traves de sus canales fisico y digital.

La empresa ofrece productos de electronica de consumo, accesorios tecnologicos y servicios de garantia extendida. Su modelo de negocio depende en gran medida de la retencion de clientes recurrentes, quienes representan el 68% de los ingresos totales.

Problema Identificado

El equipo de marketing de RetailCo ha identificado que un 23% de sus clientes activos deja de comprar cada ano sin ninguna senal de alerta previa. El costo de adquirir un nuevo cliente es 5 veces mayor que retener uno existente. La empresa pierde aproximadamente COP $480 millones anuales debido al abandono no anticipado de clientes.

Necesidad principal: Predecir con antelacion que clientes tienen alta probabilidad de abandonar la plataforma, para que el equipo de retencion pueda actuar con campanas personalizadas antes de que ocurra el abandono.

1.b. Generacion y Seleccion de Alternativas de Solucion

AlternativaDescripcionVentajasDesventajasSeleccion
A1: Desarrollo propio Modelo ML con scikit-learn en servidor propio. Control total, sin costo de plataforma. Requiere equipo de data science e infraestructura propia. Descartada
A2: Azure Cognitive Services Servicios cognitivos prediseñados de Azure. Facil de implementar. No aplica al problema de churn — son servicios de percepcion. Descartada
A3: Azure ML Designer Pipeline visual con componentes drag-and-drop para clasificacion. Visual, sin codigo, integrado con Azure, despliegue nativo REST. Requiere workspace de Azure ML. Seleccionada
A4: Azure AutoML Entrenamiento automatico de multiples modelos. Automatizado, alta precision. Mayor costo computacional, menos control. Alternativa futura
Alternativa seleccionada: A3 — Azure Machine Learning Designer. Permite construir el pipeline visualmente, entrenar un modelo de clasificacion binaria (churn / no churn), evaluar su desempeno y desplegarlo como endpoint REST, todo dentro del ecosistema de Azure.

1.c. Propuesta del Pipeline de Azure Machine Learning

Dataset

Se utilizara el dataset publico Telco Customer Churn (IBM Sample Data). Contiene 7.043 registros de clientes con 21 variables incluyendo datos de servicio, contrato, facturacion y la variable objetivo Churn (Si/No).

Componentes del Pipeline

PasoComponente Azure MLFuncion
1Dataset (Data Asset)Cargar el archivo CSV del dataset de churn en Azure ML.
2Select Columns in DatasetSeleccionar las columnas relevantes para el modelo.
3Clean Missing DataEliminar o imputar valores nulos en el dataset.
4Edit MetadataMarcar la columna "Churn" como etiqueta (variable objetivo).
5Normalize DataNormalizar variables numericas (MinMax scaling).
6Split DataDividir dataset: 80% entrenamiento / 20% prueba.
7Two-Class Boosted Decision TreeAlgoritmo de clasificacion para predecir churn.
8Train ModelEntrenar el modelo con los datos de entrenamiento.
9Score ModelAplicar el modelo entrenado a los datos de prueba.
10Evaluate ModelCalcular metricas: AUC, Accuracy, Precision, Recall.
11Real-time EndpointDesplegar el modelo como servicio web REST.
