2.c. Descripcion de los Componentes

1. Azure Blob Storage

Servicio de almacenamiento de objetos de Azure. Almacena el dataset CSV de forma persistente y segura. Es el punto de entrada de los datos al pipeline de Azure ML.

2. Azure ML Data Asset

Recurso de datos registrado en Azure Machine Learning que apunta al archivo en Blob Storage. Permite versionar los datasets y reutilizarlos en multiples experimentos.

3. Select Columns in Dataset

Filtra las columnas del dataset conservando solo las variables mas relevantes para el modelo: tenure, MonthlyCharges, TotalCharges, Contract, PaymentMethod e InternetService, entre otras.

4. Clean Missing Data

Limpia valores faltantes del dataset. Se configura para eliminar filas con valores nulos o reemplazarlos por la media o moda segun el tipo de variable.

5. Edit Metadata

Define el rol de cada columna en el pipeline. Marca la columna "Churn" como la etiqueta (variable objetivo) que el modelo debe aprender a predecir.

6. Normalize Data

Aplica normalizacion MinMax a las variables numericas continuas. Mejora la convergencia del algoritmo y evita que variables con mayor magnitud dominen el entrenamiento.

7. Split Data

Divide el dataset en dos subconjuntos: 80% para entrenamiento y 20% para evaluacion. La division es aleatoria con semilla fija para garantizar reproducibilidad.

8. Two-Class Boosted Decision Tree

Algoritmo de clasificacion binaria basado en arboles de decision con boosting. Combina multiples arboles debiles para crear un clasificador fuerte. Es robusto con datos tabulares y maneja bien el desbalance de clases.

9. Train Model

Ejecuta el entrenamiento combinando el algoritmo con los datos de entrenamiento. Produce un modelo serializado listo para hacer predicciones sobre nuevos datos.

10. Score Model

Aplica el modelo entrenado al conjunto de prueba. Para cada cliente genera una probabilidad de churn (0.0 a 1.0) y una etiqueta predicha (True/False).

11. Evaluate Model

Calcula las metricas de desempeno comparando predicciones con valores reales. Genera la curva ROC, AUC, Accuracy, Precision y Recall para validar la calidad del modelo.

12. Real-time Endpoint (Azure ML)

Despliega el modelo entrenado como servicio web REST en un contenedor de Azure. Recibe datos JSON de un cliente y retorna la prediccion de churn en tiempo real. Es consumido por el CRM de RetailCo para ejecutar campanas de retencion automatizadas.
